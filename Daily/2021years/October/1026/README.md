# debounce & throttle

자바스크립트의 특성상 작업은 콜스택에 쌓여 하나씩 실행되는 순서를 기억해 놓았다가 실행하는데 (콘솔 - 콜스택 - > 웹API - 웹요청 → 콜백큐 → 콜스택 → 실행)

이렇게 작업이 될 경우 setInterval과 같은 함수를 주었을때 넘나 비효율적이다 이를 최적화 하기 위해 나온것이 디바운스와 트로틀이다

트로틀은 초당 함수의 실행 횟수를 제한하고 디바운스는 이벤트가 끝날때까지 기다렷다가 시작된다.

따라서 뜨로틀은 일정주기로 실행하니 유저가 즉각적인 결과를 원하는 경우 적합하고 디바운스는 마지막 이벤트가 의미있는 경우 적합하다.
